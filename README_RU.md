Это **навык AI-агента (в формате SKILL.md)**. После установки достаточно говорить с ним обычным языком — «преврати этот запрос в котировку», «клиент подтвердил, сделай PI/CI/PL» — а всю работу AI выполнит сам, вам не нужно запоминать никаких команд.

## Узнаёте себя в этом еженедельном рутине?

- Клиент присылает Excel с запросом → вы вручную **переписываете** названия товаров, характеристики, количество в котировку
- Клиент подтвердил → вы вручную **переписываете те же данные ещё раз** для PI
- Товар готов к отгрузке → **переписываете снова** для CI и упаковочного листа
- В PI закралась ошибка в количестве → она **автоматически расходится** по CI, PL и таможенным документам, и приходится искать и исправлять каждую копию
- Разным клиентам нужен разный формат PI (русскому клиенту — на русском, европейскому — на английском) → приходится вести **несколько шаблонов**

**По сути, каждый документ — это просто другое оформление одних и тех же данных**, но каждый раз вы начинаете с нуля.

Решение Trade Pipeline простое: **ведите только один файл заказа, а все документы генерируются из него автоматически. Изменили что-то в одном месте — обновились все четыре документа.**

## Что он умеет

Автоматически **превращает** Excel-запрос клиента в полный комплект документов:

| Документ | Описание |
|------|------|
| 📋 **Котировка** | Шапка компании, спецификация товаров, колонка цены оставлена пустой для заполнения |
| 📄 **Проформа-инвойс (PI)** | Данные продавца/покупателя, условия оплаты, условия поставки |
| 📄 **Коммерческий инвойс (CI)** | Двуязычная (кит./англ.) шапка, отгрузочная маркировка, данные отгрузки, сумма прописью |
| 📦 **Упаковочный лист (PL)** | Разбивка по коробкам, количество коробок/вес/штук, сводка по паллетам |

**Котировка** — жёлтым выделена колонка для ввода цены, после заполнения одним нажатием данные записываются обратно в файл заказа:

![Котировка](docs/2601_quotation.png)

**Проформа-инвойс (PI)** — шапка продавца/покупателя, товарные позиции по группам, условия оплаты:

![PI](docs/2601_pi.png)

**Коммерческий инвойс (CI)** — двуязычный (кит./англ.), отгрузочная маркировка, данные отгрузки, сумма прописью (SAY):

![CI](docs/2601_ci.png)

**Упаковочный лист (PL)** — позиции по группам, количество коробок/вес коробки/количество/нетто/брутто, сводка по паллетам:

![PL](docs/2601_pl.png)

## Кому подходит

| Ваша ситуация | Подходит? |

| Нет ERP, работаете через Excel + почту + мессенджеры | ✅ **Основной сценарий, можно использовать сразу** |
| Есть 用友/金蝶/旺店通 (китайские ERP-системы), но с внешнеторговыми документами там слабо | ✅ Подходит, дополняет подготовку и вывод документов |

### Идеальный профиль пользователя

- Заказов в год: **50–500**
- Артикулов (SKU): **менее 5 000**
- Основные рынки: **Россия / Восточная Европа / Южная Америка / Ближний Восток**
- Размер команды: **1–3 менеджера ВЭД**

## Как это соотносится с ERP

Trade Pipeline **не заменяет ERP**. ERP управляет складом, финансами, дебиторской/кредиторской задолженностью, налогами — этого инструмент не делает и не собирается делать.

Он дополняет то, с чем ERP справляется плохо **до и после себя**:

| Слабое место ERP | Как это решает Trade Pipeline |
|---|---|
| Excel-запрос от клиента приходится **вносить вручную** | Автоматическое извлечение данных о товарах |
| Название клиента пишется непоследовательно (особенно часто у российских клиентов), ERP не может сопоставить | 4-уровневое нечёткое сопоставление, если не найдено — останавливается и просит выбрать вручную |
| Каждому клиенту нужен свой формат PI/CI, для изменения шаблона ERP нужно звать консультанта | Достаточно изменить один конфигурационный файл — 5 минут |
| После экспорта котировки клиент поменял порядок строк, при обратном импорте все цены сбились | В каждой строке спрятана невидимая метка, порядок строк никак не влияет на результат |
| Для номеров стандартов DIN/ISO/GOST нет отдельных полей | Автоматическое распознавание и нормализация номеров стандартов |

---

## Чем это НЕ является

- ❌ **Не ERP** — не управляет складом, финансами, налогами
- ❌ **Не онлайн-сервис** — нет аккаунтов, нет службы поддержки, все данные хранятся только на вашем компьютере
- ❌ **Не платформа для совместной работы** — лучше всего подходит для команды из 1–3 человек
- ❌ **Не «ИИ ведёт внешнюю торговлю вместо вас»** — ИИ только помогает разбирать запросы сложного формата, всё остальное — детерминированные правила
- ❌ **Не подключается напрямую к таможне** — может подготовить данные, нужные таможенному брокеру, но не может напрямую подать их в «единое окно»

## Сразу скажем: для какой отрасли это подходит

Этот инструмент я делал **для своей собственной компании**, поэтому он сделан с учётом особенностей **отрасли металлоизделий / крепежа**:

- Ценообразование **в основном по весу** (кг / тонна)
- Характеристики товаров относительно фиксированы (стандартные изделия DIN / ISO / GOST)
- Способы упаковки тоже достаточно стандартны

**Если вы тоже работаете в отрасли металлоизделий / крепежа** (болты, гайки, шайбы, стандартные изделия, проволока, трубная арматура, фланцы, подшипники, уплотнения и т. д.),
достаточно изменить данные компании и таблицу перевода — и можно использовать.

**Если у вас другая отрасль**, характеристики, способ ценообразования (за штуку / за комплект / за метр), логика веса и упаковки — всё другое,
**этот инструмент нельзя использовать «как есть»**, потребуется доработка — в основном в трёх местах:

| Что нужно изменить | Где находится |
|---|---|
| Распознавание товаров / перевод кит.-англ. | `understanding/canonicalizer.py` + таблица перевода |
| Модель ценообразования (если не по весу) | Логика определения ценообразования в `understanding/` |
| Шаблоны документов (шапки, поля, вёрстка) | Отдельные Writer-модули в `writers/` |

Фреймворк универсален, но объём адаптации зависит от сложности вашей отрасли.

> Одной фразой: **для металлоизделий/крепежа — используйте сразу, для других отраслей — используйте как базу для доработки.**

**Отрасли, где адаптация будет сложнее**: одежда (сложная матрица размеров), электронные компоненты (вложенные BOM), сырьевые товары навалом (фьючерсное ценообразование), трансграничная электронная торговля (десятки тысяч SKU) — здесь потребуется значительная доработка, будьте готовы к этому.

## Установка

<a id="use-skill"></a>
### Установка и использование через Claude Code

Если вы ещё не пользовались [Claude Code](https://claude.ai/code): это официальный инструмент Anthropic для программирования и автоматизации на базе AI — установив его один раз, вы сможете использовать его для документов, писем, обработки данных и многого другого. После установки плагина **просто говорите с ним обычным языком** — не нужно запоминать никакие команды.

**Шаг 1: клонировать репозиторий и установить основной движок**

```bash
git clone https://github.com/Dangooy/trade-pipeline-skill.git
cd trade-pipeline-skill
pip install -e .
```

**Шаг 2: загрузить плагин в Claude Code**

В этом репозитории уже есть плагин для Claude Code (`.claude-plugin/plugin.json`), в котором два навыка:

| Название навыка | Назначение |
|---|---|
| `trade-pipeline-init` | Мастер первоначальной настройки (данные компании, условия поставки, первый клиент) |
| `trade-pipeline-run` | Запуск конвейера документов / обратной записи цен |

Запустите Claude Code в папке репозитория и командой `/plugin` загрузите локальный репозиторий как плагин (при первом запуске потребуется подтвердить доверие к этой папке). После загрузки оба навыка регистрируются автоматически.

**Шаг 3: просто общайтесь**

```text
инициализировать настройки                         → запускает trade-pipeline-init, пошагово спрашивает данные компании
преврати examples/sample_inquiry.xlsx в котировку   → запускает trade-pipeline-run, автоматически проходит весь конвейер
клиент подтвердил, запиши цены и сделай PI/CI/PL    → автоматически выполняет обратную запись цен + генерирует официальные документы
```

> Навык в фоновом режиме запускает детерминированный движок для генерации документов, вам не нужно вручную вводить никакие параметры — просто скажите Claude, что нужно сделать, а он всё соберёт.

<a id="use-workbuddy"></a>
### Установка и использование через Tencent WorkBuddy

Пользователи в Китае, использующие [WorkBuddy](https://codebuddy.cn/work) (настольный AI-агент от Tencent), тоже могут использовать этот инструмент — он совместим с форматом навыков SKILL.md, достаточно скопировать папку навыка.

> **Предварительное условие**: на компьютере уже установлены Python 3.11+ и Git. Навык под капотом вызывает движок на Python, поэтому этот шаг пропустить нельзя.

**Windows (PowerShell)**

```powershell
# 1. Скачать проект и установить движок
git clone https://github.com/Dangooy/trade-pipeline-skill.git
cd trade-pipeline-skill
pip install -e .

# 2. Скопировать оба навыка в папку навыков WorkBuddy
mkdir "$env:USERPROFILE\.workbuddy\skills" -Force
Copy-Item -Recurse -Force ".\.claude\skills\trade-pipeline-init" "$env:USERPROFILE\.workbuddy\skills\"
Copy-Item -Recurse -Force ".\.claude\skills\trade-pipeline-run"  "$env:USERPROFILE\.workbuddy\skills\"
```

**Mac / Linux (Терминал)**

```bash
# 1. Скачать проект и установить движок
git clone https://github.com/Dangooy/trade-pipeline-skill.git
cd trade-pipeline-skill
pip install -e .

# 2. Скопировать оба навыка в папку навыков WorkBuddy
mkdir -p ~/.workbuddy/skills
cp -r .claude/skills/trade-pipeline-init ~/.workbuddy/skills/
cp -r .claude/skills/trade-pipeline-run  ~/.workbuddy/skills/
```

После установки достаточно вручную вызвать навык в диалоге WorkBuddy:

```text
@trade-pipeline-run преврати этот запрос в котировку
```

> Примечание: WorkBuddy работает на отечественных моделях (Hunyuan / DeepSeek / GLM и т. д.), а не на Claude. Промпты навыка написаны с расчётом на манеру понимания Claude, поэтому при смене модели иногда могут возникать отклонения в срабатывании или понимании шагов — если с первого раза не заработает как надо, это нормально, будем рады issue с обратной связью.

## Настройка перед первым использованием

Поскольку на котировке, PI и CI должны печататься данные вашей компании, перед первым использованием нужно заполнить некоторые базовые сведения.

Скажите в Claude Code **«инициализировать настройки»** (это запустит `trade-pipeline-init`), и он поочерёдно спросит вас:

| Информация | Где используется | Обязательно? |
|------|----------|--------|
| Название компании (кит./англ.) | Шапка всех документов | Да |
| Адрес | Шапка PI/CI | Да |
| Контактное лицо / email / телефон | Котировка и PI | Да |
| Банковские реквизиты | Платёжные данные в PI | Нет (можно добавить позже) |
| Условия поставки | FOB / CIF / DDP / EXW / CFR | Да |
| Валюта | USD / CNY / EUR | Да |
| Порт отгрузки | Нижний колонтитул PI/CI | Да |
| Первый клиент | Данные покупателя | Нет (можно сначала использовать временного клиента `_new`) |

> Эта информация хранится в файле `config/config.yaml`. YAML — простой текстовый формат, его можно открыть и редактировать в обычном блокноте — выглядит как список покупок, программировать не нужно.

## Повседневное использование: просто говорите с Claude

После установки плагина вам не нужно запоминать никакие команды — просто говорите Claude обычным языком, что нужно сделать. Типичные формулировки:

| Вы говорите Claude | Что он сделает |
|---|---|
| «Преврати этот запрос в котировку» (с приложенным Excel) | Разберёт запрос → сгенерирует котировку (для переговоров о цене с клиентом) |
| «Клиент подтвердил, запиши цены и сделай PI/CI/PL» | Запишет заполненные цены обратно в файл заказа → сгенерирует проформа-инвойс / коммерческий инвойс / упаковочный лист |
| «Это новый клиент, сначала сделай котировку с временными данными» | Сгенерирует котировку с временным профилем покупателя, данные клиента можно добавить позже |
| «Имя клиента не сопоставилось, помоги подтвердить» | Выведет список кандидатов на выбор, после выбора продолжит генерацию |

> **Рекомендуемый порядок работы**: сначала сделайте котировку для переговоров о цене, а после подтверждения клиентом — запишите цены и сгенерируйте официальные документы (PI/CI/PL).
> Так на этапе котировки не придётся многократно переделывать весь комплект документов.

<details>
<summary>Справочник для разработчиков: команды нижнего уровня (именно их вызывает навык под капотом)</summary>

```bash
# Этап 1: сгенерировать только котировку (для переговоров о цене с клиентом)
python -m trade_pipeline --input inquiry.xlsx --order 2601 --buyer global_fasteners --quote-only

# Этап 2: после подтверждения клиентом — записать цены + сгенерировать PI / CI / PL
python -m trade_pipeline --price-update output/2601/2601_quotation.xlsx --model output/2601/2601_model.json

# Сгенерировать весь комплект сразу (для срочных случаев или демо):
python -m trade_pipeline --input inquiry.xlsx --order 2601 --buyer global_fasteners

# Новый клиент (сначала с временным профилем, данные добавить позже):
python -m trade_pipeline --input inquiry.xlsx --order 2601 --buyer _new --quote-only

# Если система не может сопоставить имя клиента, она останавливается и создаёт review.json, после подтверждения — продолжить:
python -m trade_pipeline --input inquiry.xlsx --order 2601 --confirm output/2601/2601_review.json
```

## Как был создан этот инструмент

Я не профессиональный программист, а сотрудник отдела внешней торговли на производственном предприятии, точнее говоря — так называемый «厂二代» (наследник владельца завода). Из-за ограниченности доступных ресурсов мне очень нужен был мощный инструмент-помощник — AGENT.

Причина появления этого инструмента проста: **я каждую неделю тратил по несколько часов на ручное оформление торговых документов, и мне это надоело.** Claude Code помог превратить накопленный за годы практический опыт в работающую систему автоматизации.

Пример: в первой версии функция обратной записи цен использовала номер строки — цена из строки 3 записывалась обратно в строку 3. Технически это было абсолютно правильно. Пока однажды клиент не вставил в котировку несколько строк с заголовками групп — я запустил обратную запись, и все цены сбились. PI тоже получился с ошибками.

Я рассказал об этом баге Claude, и его первой реакцией было «исключить строки-заголовки при разборе». Я сказал, что это не сработает — невозможно предсказать, как клиент будет редактировать Excel. Мы обсуждали это в три захода, и в итоге я предложил: «А можно спрятать в каждой строке невидимый ‘паспорт’?» Claude реализовал решение со скрытой колонкой UUID.

**Такое решение не встретишь в учебниках по программированию, потому что оно родилось из реального бага в продакшене.**

> Инструменты AI-программирования наиболее сильны, когда у вас есть глубокая экспертиза в предметной области — я прекрасно знаю, как должен выглядеть PI, потому что делал их вручную сотни раз.

## Где хранятся ваши данные

Все данные о клиентах, данные заказов, сгенерированные документы **хранятся только на вашем компьютере**, ничего не загружается ни на какие серверы.

| Файл | Что хранит |
|------|--------|
| `config/config.yaml` | Данные вашей компании, данные клиентов, условия поставки (можно редактировать в обычном блокноте) |
| `output/номер_заказа/` | Файл заказа и весь комплект сгенерированных документов для каждого заказа |

Это означает:
- ✅ Данные полностью под вашим контролем
- ✅ Можно отслеживать историю изменений через Git
- ⚠️ Резервное копирование нужно делать самостоятельно (рекомендуется регулярно копировать на внешний диск или в зашифрованное облачное хранилище)
